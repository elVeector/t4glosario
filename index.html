Glosario de Comandos Git por Categoría

Configuración:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git config Configura opciones de Git en el sistema local --global para configuración global, --local para configuración de repositorio git config --global user.name "Tu Nombre"
git config --list Muestra todas las configuraciones Ninguno git config --list

Inicialización:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git init Inicializa un nuevo repositorio Git en un directorio Ninguno git init
git clone Clona un repositorio remoto en tu sistema local URL del repositorio remoto (ej. https://github.com/user/repo.git) git clone https://github.com/user/repo.git
git clone --depth 1 Clona un repositorio con la última revisión --depth para especificar la profundidad de clonación (ej. 1 para la última revisión) git clone --depth 1 https://github.com/user/repo.git

Escenario:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git add Agrega cambios al área de preparación Nombre de archivo o directorio para agregar cambios git add archivo.txt
git commit Registra cambios en el repositorio local -m para especificar un mensaje de commit git commit -m "Mensaje de commit"
git reset Desecha cambios en el área de preparación --soft para mantener los cambios en el directorio de trabajo, --hard para eliminar cambios también git reset HEAD archivo.txt
git stash Almacena temporalmente cambios no comprometidos save para almacenar los cambios con un mensaje git stash save "Mensaje"

Ramas:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git branch Muestra la lista de ramas -a para mostrar todas las ramas, -d para eliminar una rama git branch
git checkout Cambia de rama o de punto en la historia Nombre de rama o hash de commit para cambiar a una rama o punto específico git checkout nombre_rama
git merge Fusiona cambios de una rama en otra Nombre de la rama a fusionar git merge rama_a_fusionar
git rebase Reescribe la historia combinando cambios de otra rama Nombre de la rama para rebasear git rebase otra_rama
git branch -m Renombra una rama existente Nuevo nombre para la rama git branch -m nombre_nuevo
git branch -c Crea una nueva rama y se cambia a ella Nombre de la nueva rama git branch -c nueva_rama
git branch -u Establece la rama de seguimiento de una rama local rama_local rama_remota para establecer una relación de seguimiento git branch -u rama_local rama_remota

Inspeccionar y Comparar:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git status Muestra el estado actual de los archivos Ninguno git status
git log Muestra un registro de commits Varios opciones como --oneline, --graph, etc. git log --oneline
git log --graph Muestra un registro de commits con gráfico de ramas Ninguno git log --graph
git diff Muestra las diferencias entre áreas --staged para ver las diferencias en el área de preparación git diff
git blame Muestra quién modificó cada línea de un archivo Nombre del archivo git blame archivo.txt
git show Muestra información detallada sobre un commit hash_commit para mostrar información de un commit específico git show hash_commit
git reflog Muestra un registro de acciones de referencia Varios opciones como --all para ver todas las acciones git reflog

Seguimiento de los Cambios:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git add Agrega cambios al área de preparación Nombre de archivo o directorio para agregar cambios git add archivo.txt
git commit Registra cambios en el repositorio local -m para especificar un mensaje de commit git commit -m "Mensaje de commit"
git reset Desecha cambios en el área de preparación --soft para mantener los cambios en el directorio de trabajo, --hard para eliminar cambios también git reset HEAD archivo.txt
git checkout Descarta cambios no comprometidos en el directorio de trabajo Nombre de archivo para descartar cambios git checkout archivo.txt
git reset --hard Elimina todos los cambios locales y vuelve al último commit Ninguno git reset --hard

Ignorando los Patrones:

Comando Breve descripción Parámetros y descripción de los mismos Ejemplo
.gitignore Archivo para especificar patrones de archivos a ignorar Patrones de archivos o directorios que se deben ignorar Crear o editar un archivo .gitignore con patrones de archivos a ignorar
git clean Elimina archivos no rastreados en el directorio de trabajo -n para ver qué archivos se eliminarán, -f para eliminar definitivamente git clean -n, git clean -f

Compartir y Actualizar:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git remote Muestra y administra repositorios remotos add, remove, rename, set-url, show, etc. git remote add origin URL, git remote -v
git fetch Obtiene cambios del repositorio remoto sin fusionarlos Nombre del repositorio remoto (por defecto, origin) git fetch origin
git push Envía cambios locales al repositorio remoto Nombre del repositorio remoto y rama a la que se desea enviar los cambios git push origin master
git pull --rebase Obtiene cambios y aplica rebase en lugar de merge Ninguno git pull --rebase origin rama
git fetch --prune Elimina referencias remotas obsoletas Ninguno git fetch --prune
git remote add upstream Agrega un repositorio remoto adicional nombre_repositorio URL para agregar un nuevo repositorio remoto git remote add upstream URL

Tags:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git tag Muestra la lista de tags Ninguno git tag
git tag nombre_tag Crea un nuevo tag en el commit actual Nombre del tag git tag v1.0
git tag -a nombre_tag -m "Mensaje" Crea un nuevo tag anotado con un mensaje Nombre del tag y mensaje git tag -a v1.1 -m "Versión 1.1"
git show nombre_tag Muestra información sobre un tag específico Nombre del tag git show v1.0
git push origin nombre_tag Envía un tag al repositorio remoto Nombre del tag git push origin v1.0
git checkout nombre_tag Cambia a un commit asociado a un tag específico Nombre del tag git checkout v1.0
git tag -d nombre_tag Elimina un tag localmente Nombre del tag git tag -d v1.0
git push --delete origin nombre_tag Elimina un tag de un repositorio remoto Nombre del tag git push --delete origin v1.0

Reescribir la Historia:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git commit --amend Modifica el último commit --no-edit para mantener el mensaje de commit anterior git commit --amend
git rebase -i Reorganiza y edita commits en una rama -i para realizar una reorganización interactiva git rebase -i HEAD~3
git reset Mueve la rama actual a un commit específico --soft para mantener los cambios en el área de preparación, --hard para eliminar cambios también git reset HEAD~2
git reflog Muestra un registro de acciones de referencia Varios opciones como --all para ver todas las acciones git reflog

Commits Temporales:

Comando Descripción Parámetros y descripción de los mismos Ejemplo
git stash Almacena temporalmente cambios no comprometidos save para almacenar los cambios con un mensaje git stash save "Mensaje"
git stash list Muestra la lista de cambios almacenados en el stash Ninguno git stash list
git stash apply Aplica los cambios almacenados en el stash Nombre del stash (opcional) git stash apply stash@{0}
git stash pop Aplica y elimina los cambios almacenados en el stash Nombre del stash (opcional) git stash pop stash@{1}
git stash drop Elimina un stash específico Nombre del stash git stash drop stash@{2}
git stash clear Elimina todos los stashes Ninguno git stash clear

Conclusión

Los comandos de Git son una herramienta esencial para el control de versiones y la gestión de proyectos de desarrollo de software. En las tablas se ofrece una referencia rápida y completa para cada comando, sus parámetros y ejemplos de uso, en si estos comandos facilitan la creación, seguimiento y colaboración en proyectos, lo que mejora la eficiencia y la organización en el desarrollo de software, el conocer estos comandos es muy fundamental para cualquier desarrollador que trabaje con Git.
